# 内容监控工具前端原型设计

## 目标

构建一个适合内容运营人员日常使用的 Next.js 前端原型页面，用于展示按分类管理的内容监控工作台。原型重点验证信息架构、交互路径和高频操作的清晰度，不接入真实后端。

## 核心信息架构

页面采用“左侧分类管理 + 右侧分类详情”的双栏工作台结构。

- 左侧：监控分类管理
- 右侧：当前分类详情
  - 内容
  - 选题分析与报告
  - 监控设置

该结构同时满足两层操作：

- 顶层在多个监控分类之间快速切换
- 分类内完成内容查看、报告浏览和监控配置

## 交互设计

### 1. 分类管理

左侧使用卡片列表展示多个监控分类，每张卡片包含：

- 分类名称
- 监控平台数量
- 关键词与博主数量
- 最近一次运行时间
- 当前运行状态

默认选中一个分类，并在右侧即时切换详情。

### 2. 内容 Tab

内容页用于查看当前分类下采集到的平台内容。

#### 平台筛选

不用下拉框，改为平铺平台标签按钮。原因：

- 高频操作更快
- 所有平台一眼可见
- 更适合多平台切换的工作流

#### 时间选择

采用“横向日期卡片轨道”作为主要时间交互，而不是输入框或日历面板。每张卡片展示：

- 日期
- 当日采集内容数
- 爆款数
- 主平台提示

设计理由：

- 用户无需先知道具体日期
- 能用数据密度引导点击
- 比热力图更适合原型阶段的内容详情浏览

#### 内容展示

右下区域展示当天对应平台下的内容卡片，字段包含：

- 标题
- 平台
- 作者
- 发布时间
- 热度值
- 标签
- 简短摘要

默认以热度排序，突出热门内容。

### 3. 选题分析与报告 Tab

将选题分析与选题报告合并为一个工作区。

#### 日期维度

默认展示最新报告。左侧提供“报告日期卡片列表”，卡片展示：

- 日期
- 报告标题
- 一句话摘要
- 热门方向数

设计理由：

- 用户不用猜“哪一天值得看”
- 卡片摘要会主动提示内容重点
- 比空白日期选择器更有引导性

#### 选题维度

顶部增加“最近 7 天选题汇总”切换，用于跨天汇总查看所有选题方向。

#### 报告内容

报告主区域展示：

- 昨日热点总结
- AI 观察结论
- 建议重点跟进的平台与话题
- 3 到 5 个选题建议

每个选题包含：

- 方向标题
- 选题简介

### 4. 监控设置 Tab

设置页按区块展示：

- 监控平台
- 对标关键词
- 对标博主/账号
- 自动运行频率

关键词和博主使用标签与列表组合展示，强调一个分类下支持多平台、多关键词、多博主的组合监控。

## 视觉方向

整体风格简洁、专业、偏运营后台：

- 明亮浅色背景
- 深色文本与冷灰蓝主色
- 数据区块层级清晰
- 通过大面积留白和卡片化分组降低阅读负担

## 假数据策略

预置 3 个监控分类：

- ClaudeCode 选题监控
- VibeCoding 选题监控
- AI 出海内容监控

每个分类包含：

- 4 个平台
- 5 到 7 天内容数据
- 多个关键词与对标博主
- 每日报告与最近 7 天选题建议

## 技术方案

采用轻量 Next.js App Router 原型实现：

- 单页面工作台
- 本地 TypeScript 假数据
- 组件拆分为分类侧栏、顶部概览、内容工作区、报告工作区、设置工作区
- 使用少量客户端状态管理当前分类、Tab、平台与日期筛选

## 风险与边界

- 本次不接后端，不实现真实定时任务，仅用界面状态表达“每日自动运行”
- 本次不实现登录、权限与增删改弹窗，仅聚焦浏览与配置展示
- 原型优先展示产品结构与交互，不追求完整表单行为
