# 微信与小红书双平台关键词采集设计

## 目标

在现有微信公众号监控基础上，新增小红书关键词采集能力，并让顶部“关键词搜索”成为全局采集入口。

本次设计目标：

- 左侧第二项替换成“小红书监控”
- 顶部关键词搜索增加平台选择：`微信`、`小红书`、`全选`
- 用户回车后，可按选择触发微信、小红书或双平台并行采集
- 每个平台各自产生一条 SQLite 历史记录
- 小红书采集结果持久化保存到 SQLite
- 左侧历史关键词列表按“关键词 + 平台来源”分别展示
- 为未来双平台的 AI 选题分析保留统一关联结构

## 当前状态确认

当前系统仅完成了微信公众号链路：

- 微信关键词查询已接入
- 微信文章已保存到 SQLite
- 历史关键词列表已可显示微信记录
- 每次微信查询会创建一条待分析报告占位

当前系统尚未具备：

- 小红书接口接入
- 小红书结果落库
- 顶部平台选择器
- 同一关键词按来源拆分历史记录

## 搜索区交互设计

顶部搜索区改为三段式：

- 关键词输入框
- 平台选择器
- 搜索按钮

平台选项：

- `微信`
- `小红书`
- `全选`

默认值为 `全选`。

### 回车后的行为

#### 1. 选择 `微信`

- 只请求微信接口
- 只保存微信历史记录
- 左侧历史新增一条 `关键词 / 微信`

#### 2. 选择 `小红书`

- 只请求小红书接口
- 只保存小红书历史记录
- 左侧历史新增一条 `关键词 / 小红书`

#### 3. 选择 `全选`

- 并行请求微信和小红书接口
- 微信和小红书分别写入 SQLite
- 分别新增两条历史记录
  - `关键词 / 微信`
  - `关键词 / 小红书`

## 左侧结构调整

### 分类区

保留三项分类，但第二项替换：

1. 公众号文章监控
2. 小红书监控
3. AI 出海内容监控

### 历史关键词列表

每条历史卡片显示：

- 关键词
- 平台来源
- 查询时间
- 返回内容数
- 选题状态

示例：

- `网文出海 / 微信 / 3 篇 / 待分析`
- `网文出海 / 小红书 / 12 篇 / 待分析`

点击某条历史记录后，右侧按该来源展示详情。

## SQLite 设计调整

保留已存在的 `keyword_searches` 与 `topic_reports`。

### keyword_searches

继续作为统一入口，一次“关键词 + 来源平台”就是一条记录。

关键字段：

- `id`
- `keyword`
- `source_type`
  - `wechat`
  - `xiaohongshu`
- `status`
- `article_count`
- `created_at`
- `updated_at`

### wechat_articles

继续保存微信文章。

### xiaohongshu_notes

新增，用于保存小红书笔记。

建议字段：

- `id`
- `search_id`
- `note_id`
- `title`
- `desc`
- `author_name`
- `author_id`
- `author_avatar`
- `liked_count`
- `comments_count`
- `collected_count`
- `shared_count`
- `note_type`
- `tag_title`
- `cover_url`
- `publish_time`
- `publish_time_str`
- `raw_json`
- `created_at`

### topic_reports

继续复用：

- 每次微信搜索记录生成一条 `pending`
- 每次小红书搜索记录也生成一条 `pending`

### topic_suggestions

继续预留给未来 AI 分析生成。

## 数据映射设计

### 微信

维持现有映射方案不变。

### 小红书

将接口结果映射为当前页面可复用的内容卡片结构。

建议规则：

- 标题：`note.title`
- 作者：`note.user.nickname`
- 发布时间：`note.timestamp`
- 热度：`liked_count + comments_count + collected_count`
- 摘要：优先 `abstract_show`，否则 `desc`
- 标签：`tag_info.title` + `note_attributes`
- 封面：`images_list[0].url`
- 链接：当前接口未直接提供标准笔记链接，可先不展示外链

## 右侧详情区设计

点击历史记录后，右侧主区域进入“该来源的历史详情”。

### 内容 Tab

- 点击微信历史：展示微信文章结果
- 点击小红书历史：展示小红书笔记结果

### 选题分析与报告 Tab

如果当前是历史记录：

- 显示 `关键词 + 来源平台 + 待分析`
- 说明后续将基于该次来源记录生成 AI 选题分析

### 监控设置 Tab

当前为微信或小红书监控分类时，补充说明：

- 当前关键词
- 当前来源平台
- 结果会保存到 SQLite

## API 设计

### 新增 `/api/xiaohongshu-notes`

服务端调用小红书接口：

- 请求第三方接口
- 保存查询记录到 `keyword_searches`
- 保存笔记到 `xiaohongshu_notes`
- 创建 `topic_reports` 占位
- 返回 `searchId`、`keyword`、`sourceType`、`days`

### 调整现有 `/api/search-history`

继续返回统一历史记录列表，但每条记录包含来源平台。

按 `id` 获取详情时：

- 如果来源是 `wechat`，返回微信文章详情
- 如果来源是 `xiaohongshu`，返回小红书笔记详情

- 顶部平台选择器可选微信、小红书、全选
- `全选` 时会生成两条历史记录
- 左侧历史列表按“关键词 + 来源平台”显示
- 点击历史记录能按来源查看对应数据
